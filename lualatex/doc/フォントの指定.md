lualatexです。
\documentclass{ltjsarticle} と指定しています。

次のようにソースを書いて
```
\begin{spacing}{0.7}
 \begin{minted}[linenos=none]{text}
 MariaDB [rensyu]> select * from employees;
 +----+-----------------+------+-----------+------------+
 | id | name            | age  | birthyear | department |
 +----+-----------------+------+-----------+------------+
 |  1 | 菅原文太        |   40 |      1933 | 総務部     |
 |  2 | 千葉真一        |   34 |      1939 | 開発部     |
 |  4 | 梶芽衣子        |   26 |      1947 | 営業部     |
 +----+-----------------+------+-----------+------------+
 4 rows in set (0.000 sec)
 \end{minted}
\end{spacing}
```

次のように出力したいのですが、
```
+----+-----------------+-----+-----------+------------+
| id | name            | age | birthyear | department |
+----+-----------------+-----+-----------+------------+
| 1  | 菅原文太        | 40  | 1933      | 総務部     |
| 2  | 千葉真一        | 34  | 1939      | 営業部     |
+----+-----------------+-----+-----------+------------+
```
全角文字と半角文字が1行中に混在すると、
きれいに出力されません。
フォント指定を適切にすればいいのでしょうけれこ、
どんなフォントを指定すればいいか、わかりません。
何かいい方法がありますか？


LuaLaTeX + `ltjsarticle` + `minted` の組み合わせで
**全角と半角が混在した表がずれる問題**は、典型的に

> **等幅フォントでも「和文は2倍幅」になっていない**
> ことが原因です。

つまり、

* 欧文は等幅
* 日本語はプロポーショナル（または1.0幅扱い）
  になってしまい、罫線が崩れます。

---

# 結論（最も安定する方法）

**和文も欧文も「等幅」で表示するフォントを指定する**
のが一番確実です。

LuaLaTeX では次のように設定します。

```latex
\usepackage{luatexja-fontspec}

\setmonofont{DejaVu Sans Mono}
\setmonofont{Noto Sans Mono CJK JP}[
  Scale=MatchLowercase
]

\setmonofont{Noto Sans Mono CJK JP}
```

ただし実際には **和文用と欧文用を分けて指定**します。

---

# 推奨設定（実用的）

Ubuntu環境とのことなので、次が一番トラブルが少ないです。

```latex
\usepackage{luatexja-fontspec}

\setmainfont{Noto Serif}
\setsansfont{Noto Sans}

% 等幅フォント
\setmonofont{DejaVu Sans Mono}

% 日本語等幅フォント
\setmonofont{Noto Sans Mono CJK JP}
```

そして minted の表示を強制的に等幅にします。

```latex
\setminted{
  fontfamily=tt,
  fontsize=\small
}
```

# さらに重要：luatexjaで和文等幅を有効化

これが抜けると **絶対に揃いません**。

```latex
\usepackage{luatexja}
\usepackage{luatexja-fontspec}

\setmonofont{DejaVu Sans Mono}
\setmonofont{Noto Sans Mono CJK JP}

\setmonofont{Noto Sans Mono CJK JP}
```

# 正しい最小例

以下はそのままコンパイルできるサンプルです。

```latex
\documentclass{ltjsarticle}

\usepackage{luatexja}
\usepackage{luatexja-fontspec}

\setmainjfont{Noto Serif CJK JP}
\setsansjfont{Noto Sans CJK JP}
\setmonojfont{Noto Sans Mono CJK JP}

\setmonofont{DejaVu Sans Mono}

\usepackage{minted}
\usepackage{setspace}

\setminted{
  fontfamily=tt,
  fontsize=\small
}

\begin{document}

\begin{spacing}{0.8}
\begin{minted}{text}
+----+-----------------+-----+-----------+------------+
| id | name            | age | birthyear | department |
+----+-----------------+-----+-----------+------------+
| 1  | 菅原文太        | 40  | 1933      | 総務部     |
| 2  | 千葉真一        | 34  | 1939      | 営業部     |
+----+-----------------+-----+-----------+------------+
\end{minted}
\end{spacing}

\end{document}
```

# なぜこの問題が起きるか（重要な理解）

LaTeXでは

| 種類    | 幅 |
| ----- | - |
| ASCII | 1 |
| 日本語   | 2 |

で表示されるべきですが、フォントが

* CJK対応していない
* 等幅ではない

と

```
菅原文太
```

が

```
□□□■□□
```

のようにバラバラな幅で描画されます。

minted は **単なる文字列を出すだけ**なので、フォント設定がすべてです。

# まとめ

最も重要なのはこの1行です

```latex
\setmonojfont{Noto Sans Mono CJK JP}
```

これを指定しないと
**日本語が等幅にならず、絶対に揃いません。**
